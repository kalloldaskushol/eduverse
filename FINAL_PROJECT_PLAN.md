# EduVerse Final Project Plan

## Project Summary

Build a beginner-friendly Flutter app named `EduVerse` in a separate Flutter project folder named `eduverse`.

This app is a simple education app prototype. It includes:

- Splash Screen
- Authentication
- Profile Page
- CRUD for `My Questions`
- Extra feature: `Sample Courses Page`

The app should stay easy to understand and easy to explain in viva.

Use a simple beginner style:

- `StatefulWidget`
- `TextEditingController`
- `Navigator`
- `setState(() {})`
- `FutureBuilder`
- `SnackBar`
- simple Supabase helper classes
- `Card`, `ListTile`, `TextField`, `ElevatedButton`
- small reusable widgets only where useful

Avoid:

- Provider
- Riverpod
- Bloc
- complex architecture
- over-abstracted code
- advanced state management
- unnecessary null-safe patterns that look too advanced

## App Concept

EduVerse is a student-focused education app prototype.

It contains:

- static sample courses
- user login and registration
- a user profile page
- a question management section where each user can add, read, update, and delete their own questions

## Main App Sections

- Courses
- My Questions
- Profile

## Screen List

Create these screens:

1. `SplashScreen`
2. `LoginPage`
3. `RegisterPage`
4. `AuthGate`
5. `HomePage`
6. `CoursesPage`
7. `MyQuestionsPage`
8. `ProfilePage`

## Navigation Flow

1. App starts in `main.dart`
2. Supabase initializes using placeholder config values
3. `SplashScreen` appears for about 2 seconds
4. Splash screen navigates to `AuthGate`
5. `AuthGate` checks whether the user is logged in
6. If not logged in, open `LoginPage`
7. If logged in, open `HomePage`
8. `HomePage` uses `BottomNavigationBar` for:
   - Courses
   - My Questions
   - Profile

## Authentication

### Login Page

Fields:

- Email
- Password

Behavior:

- user enters email and password
- app calls Supabase sign in
- success and error should be shown with `SnackBar`

### Register Page

Fields:

- Full Name
- University Name
- Email
- Password

Behavior:

1. Create auth user with Supabase Auth
2. Read the created auth user id from `response.user.id`
3. Insert a row into `profiles`

## Home Page

`HomePage` is mainly the navigation container.

It must:

- contain a `BottomNavigationBar`
- switch between Courses, My Questions, and Profile
- fetch profile data so the Courses page can greet the user by name

Important:

- the welcome section should appear only on `CoursesPage`
- it should not appear on Questions or Profile

## Courses Page

This is the extra feature.

Courses must be static sample data written in Dart code only.
Do not use Supabase for courses.

Each course card should contain:

- title
- instructor
- price type
- description

Requirements:

- show multiple sample courses
- use `Free` and `Paid` labels with solid color styling
- `Free` can use `Colors.green`
- `Paid` can use `Colors.orange`
- include a welcome box at the top of Courses page only

## My Questions CRUD

This is the main database CRUD section.

Each logged-in user manages only their own questions.

Fields:

- title
- subject
- description

Behavior:

- new user sees an empty state
- user can add a question
- user can edit a question
- user can delete a question
- questions are fetched from Supabase using `FutureBuilder`
- after add/edit/delete, call `setState(() {})`
- `setState(() {})` rebuilds the page and fetches the updated question list again
- new questions should appear last at the bottom

UI behavior:

- show question list in `Card` / `ListTile`
- keep the `Add New Question` button at the bottom of the page

## Profile Page

Profile page is read-only.

Show:

- Full Name
- University Name
- Email
- User ID
- simple avatar icon
- Logout button

Layout requirements:

- keep the content centered
- do not make it too wide on web/desktop
- do not add profile image upload

## Supabase Tables

### Table 1: `profiles`

Columns:

- `id` uuid primary key
- `full_name` text not null
- `university_name` text not null
- `email` text not null
- `created_at` timestamptz default now()

Important logic:

- `profiles.id` must store the same value as the authenticated Supabase user id
- each user can read only their own profile row

### Table 2: `questions`

Columns:

- `id` bigint generated by default as identity primary key
- `user_id` uuid not null
- `title` text not null
- `subject` text not null
- `description` text not null
- `created_at` timestamptz default now()

Important logic:

- `user_id` stores the current authenticated user id
- each user can only see and manage their own questions

## Supabase Usage

Use:

- Supabase Auth
- Supabase database

Do not add:

- Firebase
- profile image storage
- course table
- teacher role
- admin role

- splash screen opens first
- splash waits around 2 seconds
- auth gate correctly sends user to login or home
- login works
- register works
- register inserts into `profiles`
- home page opens after login
- bottom navigation works correctly
- Courses page shows static sample courses
- welcome box shows only on Courses page
- My Questions supports add/edit/delete
- `setState(() {})` updates the CRUD page after each change
- newly added questions appear at the bottom
- Profile page shows full name, university name, email, and user id
- logout returns to login
