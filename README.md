# EduVerse

EduVerse is a beginner-friendly Flutter project based on a simple education app idea.  
It includes:

- Splash Screen
- Authentication with Supabase
- Profile Page
- CRUD for `My Questions`
- Sample Courses Page
- Shared Course Details Page

This project was kept simple on purpose so it is easier to explain in viva.

## Features

### 1. Splash Screen

- shows the `EduVerse` title
- waits about 2 seconds
- moves to the auth check page

### 2. Authentication

- login with email and password
- register with:
  - full name
  - university name
  - email
  - password
- logout from profile page

### 3. Profile Page

- shows:
  - full name
  - university name
  - email
  - user id
- data is fetched from Supabase

### 4. My Questions CRUD

Each logged-in user can:

- add a question
- read their own question list
- update a question
- delete a question

Question fields:

- title
- subject
- description

### 5. Sample Courses Page

- static sample course list
- `Free` and `Paid` badges
- course cards are clickable
- all cards open the same course details page
- the course details page receives different course data based on the selected card

## Marking Criteria Mapping

- Authentication: Login and Register with Supabase

## Supabase Setup

### 1. Create a Supabase project

Create your project in Supabase and wait until it is fully ready.

### 2. Enable Email Authentication

In Supabase:

1. Open `Authentication`
2. Open `Providers`
3. Enable `Email`
4. If you want easier testing, disable email confirmation

### 3. Create The Tables

Run this SQL in the Supabase SQL Editor:

```sql
create table if not exists profiles (
  id uuid primary key,
  full_name text not null,
  university_name text not null,
  email text not null,
  created_at timestamptz default now()
);

create table if not exists questions (
  id bigint generated by default as identity primary key,
  user_id uuid not null,
  title text not null,
  subject text not null,
  description text not null,
  created_at timestamptz default now()
);
```

## Why `profiles.id` Uses The Auth User Id

When a user registers:

1. Supabase Auth creates the account
2. Supabase gives that user a unique id
3. The app saves the same id into `profiles.id`

This helps the app fetch the correct profile of the logged-in user.

## Enable Row Level Security

Run:

```sql
alter table profiles enable row level security;
alter table questions enable row level security;
```

## RLS Policies For `profiles`

```sql
create policy "Users can read own profile"
on profiles
for select
to authenticated
using (auth.uid() = id);

create policy "Users can insert own profile"
on profiles
for insert
to authenticated
with check (auth.uid() = id);
```

## RLS Policies For `questions`

```sql
create policy "Users can read own questions"
on questions
for select
to authenticated
using (auth.uid() = user_id);

create policy "Users can insert own questions"
on questions
for insert
to authenticated
with check (auth.uid() = user_id);

create policy "Users can update own questions"
on questions
for update
to authenticated
using (auth.uid() = user_id)
with check (auth.uid() = user_id);

create policy "Users can delete own questions"
on questions
for delete
to authenticated
using (auth.uid() = user_id);
```

## App Flow

1. App starts from `main.dart`
2. Supabase initializes
3. Splash screen appears
4. Splash screen goes to `AuthGate`
5. `AuthGate` checks if the user is logged in
6. If not logged in, open `LoginPage`
7. If logged in, open `HomePage`
8. `HomePage` uses bottom navigation:
   - Courses
   - My Questions
   - Profile

## Important Implementation Notes

- Login page uses only email and password
- Register page stores extra data in `profiles`
- `Courses` is static and does not use a Supabase table
- `My Questions` is the main CRUD feature
- CRUD refresh is done using `FutureBuilder + setState(() {})`
- new questions appear at the bottom of the list
- welcome box shows only on the Courses page
- `MediaQuery` is used in login, register, and profile pages for basic responsiveness
